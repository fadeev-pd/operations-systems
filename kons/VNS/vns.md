# Подключение к Raspberry PI по протоколу VNC

Для подключения к Raspberry Pi вам понадобится следующее:

- ваш Raspberry Pi и устройство с клиентом VNC, подключенные к одной сети (например, домашней беспроводной сети или VPN)

- имя хоста или IP-адрес вашего Raspberry Pi

- действительное имя пользователя и пароль для учетной записи на вашем Raspberry Pi

`Если вы не знаете IP-адрес своего устройства, воспользуйтесь нашей инструкцией по определению IP-адреса.`

- [Скачайте TigerVNC](https://github.com/TigerVNC/tigervnc/releases). Вы можете установить последнюю версию с страницы релизов в репозитории GitHub. Нажмите на ссылку в последнем релизе и найдите бинарный файл exe; пользователям Linux следует установить jar.

На клиентском устройстве запустите TigerVNC. Windows можно запустить бинарный файл двойным щелчком мыши. В Linux установите Java с помощью sudo apt install default-jre, затем запустите java -jar VncViewer-<version>.jar, заменив <version> на версию, которую вы скачали.

- В поле «VNC-сервер» введите IP-адрес вашего Raspberry Pi. 
```192.168.129.150```  

    ![](vnc0.png)

- Нажмите кнопку «Параметры». Перейдите на вкладку «Ввод». Установите флажок «Показывать точку при отсутствии курсора», чтобы в TigerVNC всегда был виден курсор.

    ![](vnc1.png)

- если TigerVNC выдает предупреждение, нажмите кнопку «Да», чтобы разрешить доступ к Raspberry Pi.

- Введите в окно имя пользователя ```user``` и пароль ```123```

    ![аутентификация](vnc2.png)

- сделайте скриншот рабочего стола и сохраните его на свой ПК с треем

- запустите утилиту "Подключение к удаленному рабочему столу" и введите аутентификационные данные, сделайте скриншот

- Скопируйте файл скриншотов (подключений VNC и RDP) на Raspberry PI с помощью инструментария SSH в каталог по названию Вашей фамилии

- Скопируйте файл Вашего варианта с каталога ```letunov```  с Raspberry PI с помощью инструментария SSH в каталог на Ваш локальный ПК

### справка
- Чтобы скопировать файл в определенную папку, укажите путь к ней после : в команде scp. Создайте папку перед выполнением scp, так как scp не создает папки автоматически. Например, следующая команда копирует файл с именем myfile.txt в папку project/ в домашней папке пользователя:

``` scp myfile.txt <username>@<pi_ip_address>:project/ ```

### Скопируйте файлы с вашего Raspberry Pi 

- скопируйте файл по номеру своего варианта из каталога letunov/ к себе на компьютер и сохрание его в тепозитории отчета к лабораторной работе. в отчете сделайте скриншот данной операции с треем.

Синтаксис команды простой. Это выглядит так:

```scp [опции] источник назначение```

Например, чтобы скопировать файл с именем myfile.txt из домашнего каталога пользователя на Raspberry Pi в текущий каталог на другом компьютере, выполните следующую команду:

``` scp <username>@<pi_ip_address>:myfile.txt ```

### Загрузка файла на сервер (upload)
Чтобы передать файл с компьютера на сервер, укажите путь к файлу и каталог назначения: 

```scp file.txt user@remote-host:/path/to/dest/```

Команда поможет сделать копию файла file.txt в нужный каталог на сервере.

### Скачивание файла с сервера (download)
- подключиться к серверу по SSH
- Если хотите скопировать файлы с сервера на компьютер, укажите источник и папку, затем выполните команду:

```scp user@remote-host:/path/to/file.txt ./local-dir/```

После выполнения команды файл окажется в директории.
